# gakumasu_comulist

学マスのコミュ一覧の未読管理を行うWebシステム

## 開発のモチベーション

### アイマスP・いちオタクとして

- 「学園アイドルマスター」を始めてみたが、ストーリーが大量にあって 先輩Pに追いつける気がしない
- おそらくカードごとにストーリーが用意されているゲームあるあるだが、ゲーム上でのストーリーの消化率・未読状況を見るには、 良くてキャラクター別に・悪いと所持カード別に確認を強いられる
- 結果、読んだ / 読んでいないを Wikiページを参考にして独自にメモして一覧管理にしていたが、どう考えても管理が面倒
- これを ツール化して、Wikiのページを往復して調べなおすことなく「このキャラが登場するストーリーだけは追いたい」「所持カードは全部消化したい」といった好きな順で未読管理できるようにしたい

#### 将来の展望

- これをひな形に、すでにプレイしている「アイドルマスター シャイニーカラーズ(enza/Song for Prism)」でも 未読コミュを一覧管理したい。というか正直シャニの方が本命。未読が多すぎて辛い
- それ以外にも 漫画・アニメ・ゲーム あらゆるものを積みまくっているので、積んでいるものを片っ端から可視化して、優先度をつけて減らしていきたい

### エンジニアとして

- お仕事にて ながらく自前でのプログラム開発しかやってこなかった。時代に取り残されないようにAIツールを自然に使えるようになりたい。
- Webシステム開発を真面目にやった経験はほぼないに等しいので、(AI越しではあるけど) 「フロントエンド 怖くないよ」 と言える状態にはなりたい

## 開発スタイル

- すごいと噂の Cursor さんに働かせて、ちょくちょく口と手を出すスタイルでやっていきたい
- 早いところ未読管理して、見たいアイドルを優先しないと、どんどん後回しになりそうな懸念がある。
- かといって 開発ばかりでアイマス遊べないと本末転倒。
- ということで 間を取って アイマス:開発 = 6:4くらいのペースで進めます。

- 途中経過は 気が向いたら Qiita で記事に書くかも (文書を書く練習にもなりそう)
- コミュの読了状況や感想があれば note に書くかも (ツールで note と連携するのも面白いかな？)
