# ゲーム概要と用語の関係性

## ゲームの概要

このゲームでは複数のアイドルが登場する。このゲームのプレイヤーはアイドルを育てるプロデュース業務を担うという立ち位置になっている。

このゲームでは、アイドルのキャラクターを掘り下げたお話を多数読むことができる。このお話は主にプレイヤー(プロデューサー)がアイドルとのコミュニケーションしたやりとりを描いた、短編の物語である。ゲーム内の用語でこうしたお話を「コミュ」と呼称している。他の言い換えとしては「シナリオ」「ストーリー」でも通じる。

このゲームでは、アイドルの様子を描いた「カード」が販売されていて、不定期に新規のカードが追加される。カードは「プロデュース・カード」と「サポート・カード」に分類される。

## ゲーム内でお話を読める機会

ゲームの中でお話を読める機会は複数用意されている：

### (1) アイドル個別のお話

さらに以下に分類できる：

- **(1-A) 親愛度コミュ**: アイドルとの親密さにより解禁
- **(1-B) 育成シナリオ**: アイドルを育成する過程で発生するお話
- **(1-C) プロデュース・カード・コミュ**: プレイヤーが所持している「プロデュース・カード」に紐づくお話
- **(1-D) その他コミュ**: アイドルの誕生日記念などの分類が難しいもの

### (2) サポート・カード・コミュ

プレイヤーが所持している「サポート・カード」に紐づくお話

### (3) 初星コミュ (はつぼし コミュ)

ゲーム全体で用意されているお話である

### (4) イベントコミュ

ゲーム内で定期的開催されるオンラインイベントに紐づくイベントコミュ

## このツールで実現したいこと

- 多数のコミュがあり、どれが読んでいてどれが未読の話かどうか管理したい
- コミュの未読管理は特に **(1-C)** と **(2)** を優先して管理したい。この両者はゲーム内でカード個別に読了状況を確認する必要があり、非常に煩雑
- その後 **(1-A)**、**(1-B)**、**(3)**、**(4)** はゲーム内で簡単に確認できるので優先度は下がる。一旦のツール完成後に機能追加したいので、その余地を残すこと（カードに紐づかないお話があることを考慮する）
- **(1-D)** は極めて数が少ないため対象外にする

## カードとお話の関係性 (Entity-Relationship)

※用語はゲーム内で通じる名称にしている。ビジネスロジックでは基本的にこの名称で扱いたい

### エンティティの関係性

- ゲーム内には複数の「Idol」が存在する (1:1..*)
- ゲーム内には複数の「IdolCard」が存在して、ゲームプレイヤーである「Producer」は所持していない場合がある (1:0..*)
- 「IdolCard」には対象とする「Idol」が1人以上いて属性として持っている
- 「Idol」ごとに多数の「IdolCard」が用意されている (1..*:1..*)
- 「IdolCard」は「ProduceCard」「SupportCard」の2つに分類される
- 「ProduceCard」では対象の「Idol」を必ず1人に特定できる(1:1)
- 「SupportCard」では主の「Idol」の他に、登場人物として他の「Idol」も複数人出てくることがある (1:1..*)
- 全ての「IdolCard」には「レアリティ」という属性があり、この属性は「SSR」「SR」「R」の3種類が存在している (1:1)

### ストーリーの分類

- 「Story」にはいくつか種類があるが、現時点では**(1-C)**の「ProduceCardStory」と**(2)**の「SupportCardStory」の2種類に限定して考える（※他のStoryを定義する余地を残す）

### ProduceCardStoryの詳細な分類

- 「ProduceCard」で「SSR」属性の場合、3種類の「ProduceCardStory」を持っている(1:3)
- 「ProduceCard」で「SR」属性の場合、「ProduceCardStory」は存在しない(1:0)
- 「ProduceCard」で「R」属性の場合、「ProduceCardStory」は存在しない(1:0)

### SupportCardStoryの詳細な分類

- 「SupportCard」で「SSR」属性の場合、3種類の「SupportCardStory」を持っている(1:3)
- 「SupportCard」で「SR」属性の場合、2種類の「SupportCardStory」を持っている(1:2)
- 「SupportCard」で「R」属性の場合、2種類の「SupportCardStory」を持っている(1:2)

## 用語集

### コミュ / シナリオ / ストーリー

ゲーム内でアイドルとのコミュニケーションを描いた短編の物語。このツールでは主に「ストーリー」という用語を使用するが、「コミュ」「シナリオ」でも通じる。

### プロデュース・カード (ProduceCard)

アイドルの様子を描いたカードの一種。1人のアイドルに紐づく。レアリティがSSRの場合のみ、3話のストーリーを持つ。

### サポート・カード (SupportCard)

アイドルの様子を描いたカードの一種。主のアイドルの他に、登場人物として他のアイドルも複数人出てくることがある。レアリティに応じてストーリー数が異なる（SSR=3話、SR・R=2話）。

### レアリティ (Rarity)

カードの希少度を表す属性。「SSR」「SR」「R」の3種類が存在する。

### アイドル (Idol)

ゲーム内に登場するキャラクター。複数のカードが用意されている。

### プロデューサー (Producer)

ゲームのプレイヤー。アイドルを育てるプロデュース業務を担う。
