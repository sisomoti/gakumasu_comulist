# Cursor学習レポート

## プロジェクト概要

学マス（学園アイドルマスター）のコミュニケーションストーリーの未読管理を行うWebアプリケーションを開発。Vue.js 3 + TypeScript + Viteで実装し、約2週間で39のソースファイル（約1,777行）、16のテストファイル、382のテストケースを実装。

## Vibe Cording（AI任せ）との比較：工夫されている点

### 設計・アーキテクチャの明確化

AI任せではなく、**レイヤードアーキテクチャを採用**し、Domain Model Layer、Domain Service Layerを明確に分離。**SOLID原則を適用**し、インターフェースベース設計（IDataSource、IStorageService）により高レベルモジュールが低レベルモジュールに直接依存しない設計を実現。設計判断はdata-model.md、plan.mdで明文化。

### テスト駆動開発（TDD）の実践

**テストファースト**で実装を進め、ロジック層の結合テストを追加。382のテストケースで品質を担保。

### 開発プロセスの自動化

**Pre-commit hook**（Husky + lint-staged）でMarkdown/コードの自動フォーマット。
**CI/CD**（GitHub Actions）でテストとビルドを自動実行。

### 設計の見直しとリファクタリング

初期設計が実装に適さないと判断した際、**データ設計をやり直し**。ComposableのLocalStorageへのリファクタ、共通ヘルパー関数への変更など、**段階的なリファクタリング**を実施。

## Cursorの使い方の学習ポイント

**設計の見直しを恐れない**姿勢が重要。データ設計のやり直しなど、必要に応じて設計を改善。**段階的なリファクタリング**で既存コードを改善し、**Mermaid図でアーキテクチャを可視化**して設計判断を記録。

## AIの視点：さらなる活用の余地（課題と改善点）

現在の開発では、**Plan mode**を中心に活用し、実装前の設計検討を重視しているが、**より高度な機能の活用余地**がある。例えば、**Composer**を使った大規模なリファクタリングや、**マルチファイル編集**による一貫性のある変更など。また、実装時の細かい調整や質問には**Chat**を活用し、設計と実装の両面でCursorを効果的に使い分けることで、さらに効率的な開発が可能。
